# Contributing

Thanks for improving this repository.

## Ground Rules

- Keep changes tightly scoped and easy to review.
- Update docs, screenshots, and setup steps whenever behavior changes.
- Prefer clear commit prefixes such as `feat:`, `fix:`, `docs:`, `refactor:`, and `perf:`.
- Avoid committing generated caches, secrets, or machine-specific files.

## Before Opening a Pull Request

1. Read `README.md` and follow the primary setup path.
2. Run the repository's documented validation steps.
3. Include screenshots for visible UI changes when applicable.
4. Explain the user-facing or maintainer-facing impact clearly.
